# LaboCore（ラボコア）要件定義書

**ほほ笑みラボ 教室運営システム — フェーズ1: 出欠・月謝管理**

## 1. プロジェクト概要

### 1.1 目的

シニア向けパソコン・スマホ教室「ほほ笑みラボ」の運営を支える基幹システム。
フェーズ1では出欠記録・月謝計算・生徒台帳を扱い、
月末に「誰が何回出席したか → 請求額いくらか」が一目でわかる状態にする。
プロジェクト名は LaboCore（リポジトリ名: `labocore`）。

### 1.2 背景・料金体系

- 授業は **1回ごとの従量課金制**（出席1回 × 生徒ごとの単価）
- 基本単価は 2,000円/回（例: 月4回出席 = 8,000円、3回 = 6,000円）
- 金曜クラスなど隔週の生徒は 2,500円/回 に設定予定
  → **単価は生徒ごとに個別設定できること（全体一律ではない）**
- 振替制度なし。休んだ分は請求しないだけ
- 支払いは **月末締め・出席回数分の後払い**（締めはカレンダー月: 1日〜末日）

### 1.3 フェーズ計画

| フェーズ | 内容 |
|---|---|
| フェーズ1（本書） | 出欠記録・月次集計・生徒/コマ/休講日管理 |
| フェーズ2（未定） | 請求書/領収書PDF、事前欠席連絡、生徒向け閲覧など候補 |

## 2. 利用者・利用シーン

- **利用者は管理者（まさゆき）1名のみ**。生徒・スタッフは使わない
- 利用端末は **スマホ（授業中・直後のサッと記録）と PC（1日の終わりのまとめ記録）** の両方
  → レスポンシブ必須。スマホではワンタップ中心の大きめUI
- 認証: Supabase Auth（メール+パスワード）による単一管理者ログイン
  - 新規登録画面は不要（Supabaseダッシュボードから管理者を1名作成）

## 3. 運用ルール（ヒアリング確定事項）

| 項目 | 決定内容 |
|---|---|
| 授業形態 | 曜日・時間固定のコマ制 |
| 生徒とコマ | 1人1コマ固定（水曜の人はずっと水曜） |
| 生徒数 | 10〜30人規模 |
| 振替 | なし |
| 事前欠席連絡 | 記録不要（当日の出欠のみ） |
| 休講日 | あり（祝日・先生都合）→ 登録できること |
| 過去データ | 移行なし。利用開始月から新規記録 |
| 請求額の用途 | 画面で確認できればOK（PDF出力はフェーズ2） |

## 4. 機能要件

### 4.1 今日の出欠記録（ホーム画面・最重要）

- ログイン後の初期画面。**「今日のコマ」の生徒一覧が自動表示**される
  - 今日が水曜なら水曜コマの生徒だけが並ぶ
  - 今日が休講日の場合はその旨を表示
- 生徒ごとに「出席」「欠席」をワンタップで記録
  - タップで即保存（保存ボタン不要）、視覚的フィードバックあり
  - 誤タップに備え、再タップで切り替え・取り消し可能
- 未記録の生徒が残っている場合はひと目でわかる表示
- **イレギュラー対応**: 「別の日に来た生徒」を追加できる（生徒検索 → 本日の出席として追加）

### 4.2 出欠カレンダー / 履歴

- 月表示カレンダーで日ごとの記録状況を確認
- 日付タップでその日の出欠一覧を表示・修正可能（過去日の修正OK）
- 生徒詳細画面から個人の出欠履歴（月ごとの出席回数）を確認できる

### 4.3 月次集計（請求額確認）

- 月を選択すると、生徒ごとに以下を一覧表示:
  - 出席回数 / 欠席回数
  - 請求額 = 出席回数 × その記録時点の単価
  - 全体の合計金額（当月の売上見込み）
- 支払いチェック: 生徒ごとに「支払い済み」フラグを付けられる（集金管理用）
- 前月・翌月への切り替えが簡単にできること

### 4.4 生徒管理（生徒台帳）

- 生徒のCRUD
  - 基本: 氏名、ふりがな、所属コマ、**単価（デフォルト2,000円、個別変更可）**
  - 連絡先: 電話、メール、住所（すべて任意）
  - 生年月日（任意。入会日は管理しない）
  - 使用スマホ: **Android / iPhone の区分のみ**（授業サポート時の参照用。機種名まではメモで対応）
  - 緊急連絡先: 氏名・続柄・電話（ご家族など、任意）
  - 自由メモ欄（1フィールドのみ。授業記録・つまずきの時系列管理はフェーズ2の
    「つまずきカルテ」連携に委ねるため、本システムでは持たない）
- **生徒詳細画面**: 台帳情報と出欠履歴（月ごとの出席回数・請求額）を1画面で確認できる
- 一覧はふりがな順。名前・ふりがなで検索可能
- 退会処理は論理削除（`is_active = false`）。過去の出欠・請求履歴は保持
- 単価変更時、**過去の記録の金額は変わらない**こと（後述のスナップショット方式）

### 4.5 コマ（クラス）管理

- コマのCRUD: コマ名、曜日、開始/終了時間
- 例: 「水曜午前クラス（水 10:00-12:00）」「金曜隔週クラス（金 10:00-12:00）」
- コマの廃止も論理削除

### 4.6 休講日管理

- 休講日の登録・削除（日付＋理由メモ）
- 休講日はホーム画面・カレンダーに反映（出欠記録の対象外として表示）

## 5. データモデル（Supabase / PostgreSQL）

```
classes（コマ）
- id            uuid PK
- name          text        -- 「水曜午前クラス」
- weekday       int         -- 0=日〜6=土
- start_time    time
- end_time      time
- is_active     boolean default true
- created_at    timestamptz

students（生徒）
- id            uuid PK
- name          text
- kana          text
- class_id      uuid FK → classes.id
- unit_price    int default 2000   -- 生徒ごとの1回単価
-- ▼ 生徒台帳情報（すべて任意入力）
- phone         text
- email         text
- address       text
- birth_date    date
- smartphone_os text  -- 'android' | 'iphone' | null（機種名まで必要な場合はnoteへ）
- emergency_contact_name      text  -- 緊急連絡先（ご家族など）
- emergency_contact_relation  text  -- 続柄
- emergency_contact_phone     text
- note          text  -- 自由メモ（授業記録はフェーズ2のつまずきカルテ連携に委ねる）
- is_active     boolean default true
- created_at    timestamptz

attendance_records（出欠記録）
- id            uuid PK
- student_id    uuid FK → students.id
- lesson_date   date
- status        text  -- 'present' | 'absent'
- unit_price_at_time  int   -- 記録時点の単価スナップショット
- memo          text
- created_at    timestamptz
- UNIQUE(student_id, lesson_date)

closed_days（休講日）
- id            uuid PK
- closed_date   date UNIQUE
- reason        text

payments（月次支払いチェック）
- id            uuid PK
- student_id    uuid FK → students.id
- target_month  text  -- 'YYYY-MM'
- is_paid       boolean default false
- paid_at       timestamptz
- UNIQUE(student_id, target_month)
```

### 設計上のポイント

- **単価スナップショット方式**: 出席記録時に `unit_price_at_time` へ現在単価をコピー。
  請求額は常に `SUM(unit_price_at_time) WHERE status='present'` で算出するため、
  途中で単価を変更しても過去の請求額が変わらない
- 月次集計はビュー or クエリで毎回導出（集計テーブルは持たない）
- RLS: 全テーブル「認証済みユーザーのみ全操作可」のシンプル構成（管理者1名のため）

|---|---|---|
| 1 | ログイン | メール+パスワード |

## 8. 非機能要件

- スマホ・PC両対応のレスポンシブ（スマホ幅 375px〜を想定）
- 出欠タップは楽観的更新で体感即時（失敗時はトースト+ロールバック）
- 教室のWi-Fi環境で利用。オフライン対応はフェーズ1では不要
- 10〜30人規模のためパフォーマンスチューニングは不要（Supabase無料枠で運用）
- **個人情報の取り扱い**: 生徒の連絡先・緊急連絡先を保持するため、
  RLSを全テーブルで必ず有効化し、anonキーからの未認証アクセスを遮断すること。
  共有端末でのログイン状態放置に注意（セッション有効期限はデフォルト運用でOK）

## 9. フェーズ1のスコープ外（フェーズ2候補）

- 請求書・領収書のPDF出力

- 生徒・家族向けの出席閲覧ページ
- LINE等での月謝通知
- 過去データ（紙・Excel）の取り込み
- つまずきカルテとの連携（生徒マスタの共通化）
