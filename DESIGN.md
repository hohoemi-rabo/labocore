# LaboCore（ラボコア）DESIGN.md

**ほほ笑みラボ 教室運営システム — フェーズ1: 出欠・月謝管理**

> ベース: getdesign.md「Apple design analysis」を管理ツール用に翻訳したもの。
> コンセプトは **「モノクロラグジュアリーの業務ツール」**。装飾ゼロ、余白と面の切り替えだけで
> 高級感を出す。アクセントカラーはAction Blue 1色のみ。影は原則使わない。
> ワードマークは欧文「LaboCore」（Inter 600、tracking -0.02em）。ロゴ画像は作らず文字だけで通す。

## 0. デザイン原則（最重要・全画面共通）

1. **アクセントは Action Blue（#0066cc）ただ1色。** リンク・主ボタン・選択状態・フォーカス、
   すべての「押せる」シグナルはこの色。第2のアクセント色を絶対に追加しない。
   赤・緑などのセマンティックカラーも使わない（出欠状態の表現は §4 参照）。
2. **影は使わない。** カード・ボタン・テキストにbox-shadowを付けない。
   階層は (a) 面の色の切り替え（白 ↔ #f5f5f7 ↔ ダークタイル）と (b) 1pxヘアラインで作る。
   例外は sticky バーの `backdrop-filter: blur(20px) saturate(180%)` のみ。
3. **グラデーション禁止。装飾禁止。** 雰囲気は余白とタイポグラフィで出す。
4. **押した感は `transform: scale(0.95)`。** 全ボタン・全タップ要素共通のマイクロインタラクション
   （`active:scale-95 transition-transform`）。hoverエフェクトは控えめに（色変化のみ）。
5. **ピル形状（rounded-full）は「アクション」の文法。** 主ボタン・出欠トグル・検索入力のみに使う。
   情報カードは 18px 角丸、ユーティリティボタンは 8px。中間の値を混ぜない。
6. **数字が主役。** 出席回数と金額はこのアプリの「プロダクト写真」に相当する。
   大きく・タイトに・tabular-numsで美しく見せる。

## 1. カラートークン

```
colors:
  primary:          "#0066cc"   # Action Blue — 唯一のアクセント
  primary-focus:    "#0071e3"   # フォーカスリング
  primary-on-dark:  "#2997ff"   # ダークタイル上のリンク/アクセント
  ink:              "#1d1d1f"   # 見出し・本文（純黒でなく near-black）
  ink-muted-80:     "#333333"
  ink-muted-48:     "#7a7a7a"   # 補助テキスト・disabled
  body-muted:       "#cccccc"   # ダークタイル上のセカンダリ文字
  hairline:         "#e0e0e0"   # カード枠線（1px）
  divider-soft:     "#f0f0f0"   # さらに薄い区切り
  canvas:           "#ffffff"   # 基本背景
  canvas-parchment: "#f5f5f7"   # セカンダリ背景・sticky バー（80%+blur）
  surface-pearl:    "#fafafc"
  surface-tile-1:   "#272729"   # ダークタイル（集計ヒーロー等）
  surface-tile-2:   "#2a2a2c"
  surface-black:    "#000000"   # グローバルヘッダーのみ
  on-dark:          "#ffffff"
```

Tailwind実装: 上記を `tailwind.config` の `theme.extend.colors` に登録し、
インラインhexは書かない。

## 2. タイポグラフィ

### フォントスタック（日本語対応）

```css
font-family: -apple-system, BlinkMacSystemFont, "SF Pro Text",
             "Hiragino Sans", "Noto Sans JP", system-ui, sans-serif;
```

- Apple端末では SF Pro + ヒラギノ、その他では Inter + Noto Sans JP に解決させる。
  `next/font` で **Inter（欧文・数字）+ Noto Sans JP（和文）** を読み込むこと。
- **数字・金額は必ず `font-variant-numeric: tabular-nums`**（Tailwind: `tabular-nums`）。
  集計表の桁が縦に揃うことが品質の生命線。
- 金額のカンマ区切りは `toLocaleString('ja-JP')`。「¥」は数字より一回り小さく・muted。

### スケール

| 用途 | サイズ/ウェイト | 備考 |
|---|---|---|
| ヒーロー数値（今月売上・請求額合計） | 56px / 600 / tracking -0.02em | 集計画面の主役。モバイルは40px |
| 画面タイトル | 34px / 600 / tracking -0.02em | モバイル28px |
| セクション見出し | 21px / 600 | |
| 本文 | **17px / 400 / line-height 1.47** | 16pxでなく17px。Appleの読み心地 |
| 強調本文 | 17px / 600 | |
| キャプション・ラベル | 14px / 400（強調は600） | |
| 微細情報 | 12px / 400 | |

- ウェイトの梯子は **400 / 600 のみ**（見出しに700・中間の500は使わない）。
- 和文にネガティブトラッキングを強くかけない。大見出しの `-0.02em` が上限、本文は `0`。

## 3. 面・形・余白

```
rounded:  sm 8px（ユーティリティボタン） / lg 18px（カード） / pill 9999px（アクション）
spacing:  4 / 8 / 12 / 16 / 24 / 32 / 48 / セクション64（PC）・40（モバイル）
```

- カードは `bg-canvas + border border-hairline + rounded-[18px] + p-6`。影なし。
- セクションの区切りは **面の色の切り替え**（白 ↔ parchment）で作る。線を引かない。
- コンテンツ最大幅: 一覧・集計系は 980px、センター寄せ。

## 4. 出欠ステータスの表現（本プロジェクトの核心）

セマンティックカラー（緑=出席、赤=欠席）は使わない。単一アクセント原則で表現する:

| 状態 | 見た目 |
|---|---|
| 出席 | **Action Blue塗りピル**・白文字「出席」＋チェックアイコン |
| 欠席 | **ink（#1d1d1f）塗りピル**・白文字「欠席」 |
| 未記録 | ゴーストピル（透明・1px hairline枠・ink-muted-48文字） |

- 記録済み/未記録が塗りの有無で一目でわかり、出席（青）だけが画面で「光る」。
- 出欠トグルは同一ピル内の2セグメント（出席｜欠席）。タップで即保存＋`scale(0.95)`。
- 支払い済みチェックも同文法: 未払い=ゴースト、支払い済み=Action Blue塗り＋チェック。

## 5. 画面別デザイン指針

### 5.1 グローバルナビ
- **PC**: 高さ44pxの純黒（#000000）バーを最上部に固定。左に「LaboCore」ワードマーク
  （12px/400/白・静かに）。純黒はここにしか使わない。
- **モバイル**: 下部タブバー（今日 / カレンダー / 集計 / 設定）。
  `bg-canvas-parchment/80 + backdrop-blur` のフロストガラス。アクティブタブのみAction Blue。

### 5.2 今日の出欠（ホーム）
- 上部に日付と曜日クラス名を画面タイトルで大きく（例「7月22日（水）」/ サブに「水曜午前クラス」）。
- 生徒は1人1行のリストカード。行高さ64px以上、左に氏名（17px/600）、右に出欠トグルピル。
- 全員記録済みになったら行リスト下に静かな完了表示（Action Blueのチェック＋「本日の記録完了」）。
- 休講日はparchment面に中央寄せの大きな「本日は休講日」表示のみ。何も並べない。
- 「別の日に来た生徒を追加」はゴーストピルのセカンダリボタンとして最下部に。

### 5.3 月次集計（このアプリのヒーロー画面）
- **最上部はダークタイル（#272729）のフルブリードヒーロー**: 「2026年7月」ラベル（body-muted）
  → 請求額合計を56px白のヒーロー数値 → その下に「出席のべ◯回」を14px muted。
  月送り矢印はヒーロー内に白の circular icon button（44px）。
- ヒーロー直下は白面の生徒別テーブル: 氏名 / 出席回数 / 請求額 / 支払いチェック。
  金額は右揃え・tabular-nums。行区切りは divider-soft の1pxのみ。
- ダークタイル上のリンク・アクセントは `primary-on-dark`（#2997ff）を使うこと。

### 5.4 カレンダー
- 月表示。日付セルは装飾なし、記録がある日に小さなAction Blueドット、休講日はmutedドット＋取り消し表示。
- 日付タップでシート（モバイル: 下から / PC: 右パネル）を開き、その日の出欠一覧を表示・修正。

### 5.5 生徒・コマ・休講日管理（設定配下）
- store-utility-card文法: 白カード＋hairline枠＋18px角丸のリスト。
- 追加ボタンはAction Blueピル、編集はテキストリンク（Action Blue）。
- フォーム入力: 高さ44px・pill形状・`focus:ring-2 ring-primary-focus`。ラベルは14px/600。
- 削除（退会・休講取消）だけは確認ダイアログ必須。破壊的操作もink塗りボタンで表現（赤を使わない）。

## 6. インタラクション

- タップ最小 44×44px。出欠行は64px高で誤タップを防ぐ。
- 出欠タップは楽観的更新: 即座に塗り反転 → 失敗時のみトーストでロールバック通知。
- トースト: ink塗り・白文字・pill・下部中央。3秒で消える。
- ページ遷移・リスト描画に派手なアニメーション不要。`scale(0.95)` と色のトランジション
  （150ms ease）だけで統一する。
- スケルトンよりも高速表示を優先（データ量が小さいため）。

## 7. Do / Don't

**Do**
- 迷ったら「余白を増やす」「面を切り替える」。装飾を足さない。
- 金額・回数は常に tabular-nums・右揃え。
- ボタンpaddingは 11px×22px（主）/ 8px×15px（ユーティリティ）を守る。

**Don't**
- 第2アクセント・グラデーション・カードの影・絵文字アイコンの多用。
- 出欠に赤/緑を使うこと。
- 角丸の中間値（8 / 18 / pill 以外）を混ぜること。
- 本文を16px以下にすること（微細情報を除く）。
