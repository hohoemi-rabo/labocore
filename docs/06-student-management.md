# 06. 生徒管理（生徒台帳）

**依存**: 02, 04, 05
**参照**: REQUIREMENTS.md §4.4 / DESIGN.md §5.5 / CLAUDE.md「ドメイン上の不変条件」

## 目的

生徒台帳の CRUD を実装する。単価の個別設定と論理削除がポイント。

## Todo

- [ ] 一覧 `/settings/students`: ふりがな順で表示（`is_active = true` のみ）。名前・ふりがなの部分一致検索（検索入力は pill 形状）
- [ ] 登録フォーム（必須項目）: 氏名 / ふりがな / 所属コマ（active なコマのセレクト）/ 単価（デフォルト 2,000円・個別変更可）
- [ ] 登録フォーム（任意項目）: 電話 / メール / 住所 / 生年月日 / 使用スマホ（Android・iPhone の2択+未選択）/ 緊急連絡先（氏名・続柄・電話）/ 自由メモ（1フィールドのみ。授業記録の時系列管理は持たない）
- [ ] 編集フォーム: 同項目。単価変更時に過去の `attendance_records` は一切更新しない（スナップショット方式のため触らない）
- [ ] Server Action + Zod で検証する（ラベルは 14px/600、入力は44px pill）
- [ ] 退会 = 論理削除（`is_active = false`）。確認ダイアログ必須・ink 塗りボタン。過去の出欠・請求履歴は保持される
- [ ] 変更後は `revalidatePath` で一覧を更新する

## 完了条件

- 生徒を登録・検索・編集・退会でき、退会後も過去データが残る
- 単価 2,500円/回 の生徒（金曜隔週クラス想定）を個別設定できる
