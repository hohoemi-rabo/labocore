# 03. 認証（管理者ログイン）

**依存**: 01, 02
**参照**: REQUIREMENTS.md §2・§8 / DESIGN.md §5.5（フォーム文法）

## 目的

Supabase Auth による単一管理者のメール+パスワードログインを実装し、全画面を認証必須にする。

## Todo

- [ ] ログイン画面 `/login` を作成する（メール+パスワード。入力は高さ44px・pill 形状・`focus:ring-2 ring-primary-focus`、送信は Action Blue ピルボタン）
- [ ] Server Action で `signInWithPassword` を実装する。失敗時は日本語のエラーメッセージを表示（原因の詳細は出しすぎない）
- [ ] `middleware.ts` を実装する: `@supabase/ssr` のパターンでセッションをリフレッシュし、未認証は `/login` へリダイレクト、認証済みが `/login` に来たら `/` へリダイレクト
- [ ] ログアウト動線を用意する（設定画面配下でよい）
- [ ] 新規登録画面は作らない。管理者ユーザーの作成手順（Supabase ダッシュボードから1名作成）を README に記載する

## 完了条件

- 未認証では全ページが `/login` にリダイレクトされる
- ログイン成功でホームへ遷移し、リロードしてもセッションが維持される
