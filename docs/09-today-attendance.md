# 09. 今日の出欠記録（ホーム画面・最重要）

**依存**: 03, 04, 05, 06, 08
**参照**: REQUIREMENTS.md §4.1・§8 / DESIGN.md §4・§5.2・§6 / CLAUDE.md「ドメイン上の不変条件」

## 目的

ログイン後の初期画面。今日のコマの生徒にワンタップで出欠を記録する、このアプリの日常運用の中心。

## Todo

- [x] ホーム `/`: 上部に日付と曜日を画面タイトルで大きく表示（例「7月22日（水）」）、サブにコマ名（例「水曜午前クラス」）
- [x] 今日が休講日（`closed_days` に存在）の場合: parchment 面に中央寄せの大きな「本日は休講日」表示のみ。生徒リストは出さない
- [x] 今日の曜日に一致する active コマの在籍生徒を一覧表示する（1人1行のリストカード、行高64px以上、左に氏名 17px/600、右に出欠トグルピル）
- [x] 出欠トグル: 同一ピル内の2セグメント（出席｜欠席）。未記録=ゴースト、出席=Action Blue 塗り+白文字+チェック、欠席=ink 塗り+白文字。タップで即保存（保存ボタンなし）+ `active:scale-95`
- [x] 記録の Server Action: `attendance_records` へ upsert（`UNIQUE(student_id, lesson_date)`）。**保存時に生徒の現在単価を `unit_price_at_time` へスナップショット**する
- [x] 再タップ挙動: 同じセグメントの再タップで記録を取り消し（レコード削除→未記録に戻る）、逆セグメントのタップで出席⇔欠席を切り替え
- [x] `useOptimistic` で楽観的更新: タップ即時に塗りを反転 → 失敗時のみトースト（ink 塗り・白文字・pill・下部中央・3秒）でロールバック通知
- [x] 未記録の生徒が残っているとひと目でわかる表示。全員記録済みになったらリスト下に静かな完了表示（Action Blue のチェック+「本日の記録完了」）
- [x] イレギュラー対応「別の日に来た生徒を追加」: 最下部のゴーストピルから生徒検索 → 本日の出席として追加（他コマの生徒も対象）

## 完了条件

- スマホで開いて、今日のコマの生徒全員の出欠をワンタップずつで記録し、取り消し・切り替えもできる
- 記録された行の `unit_price_at_time` にその生徒の現在単価が入っている
